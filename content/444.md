## 描述

<aside class="warning"><strong>非官方：</strong> 此 HTTP 状态码未由任何 RFC 指定。</aside>

HTTP 响应状态码 `444 No Response` 是一个非官方的 HTTP 状态码，特定于 nginx，它表示 HTTP 连接已关闭。没有任何信息，甚至包括此状态码，也不会发送到客户端。

当生成 `444 No Response` 状态码时，服务器不向客户端返回任何信息并关闭 HTTP 连接。此错误消息可以在 nginx 日志中找到，不会发送到客户端。它对于处理恶意的 HTTP 请求（例如包含非法的 `Host` 标头的请求）很有用。
