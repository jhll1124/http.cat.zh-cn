## 描述

HTTP `200 OK` 成功状态响应码表示请求已成功。默认情况下，200 响应是可缓存的。

成功的含义取决于 HTTP 请求方法：

- `GET`: 资源已被获取并在消息体中传输。
- `HEAD`: 表示性标头包含在响应中，但没有任何消息体
- `POST`: 描述操作结果的资源在消息体中传输
- `TRACE`: 消息体包含服务器接收到的请求消息。

`PUT` 或 `DELETE` 的成功结果通常不是 `200 OK`，而是 `204 No Content`（或者当资源首次上传时为 `201 Created`）。
